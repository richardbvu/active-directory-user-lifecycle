Scenario: RDP Login Failure on Domain Workstation

Environment
- Windows 11 VM joined to corp.test domain
- Managed with Active Directory and Group Policy

Problem
RDP connection failed with authorization error.

Troubleshooting Steps
1. Verified Remote Desktop was enabled on the workstation.
2. Checked Remote Desktop Users group membership.
3. Reviewed GPO setting: "Allow log on through Remote Desktop Services".
4. Verified OU placement and GPO linkage.
5. Ran gpupdate /force and gpresult to confirm policy application.
6. Retested RDP connection with manually entered credentials.

Root Cause
Cached credentials in the RDP client caused authentication failure.

Resolution
Reconnected with fresh credentials and verified successful login.

Evidence
See:
02-Troubleshooting-Evidence/Ticket-RDP-Login-Failure

I had an RDP login failure on a domain-joined Windows 11 VM in my lab environment. The error indicated that the account wasn’t authorized for remote login.

I started by verifying the basic prerequisites: I confirmed Remote Desktop was enabled on the machine and checked the Remote Desktop Users group using net localgroup "Remote Desktop Users".

Then I reviewed the Group Policy configuration controlling RDP access, specifically the Allow log on through Remote Desktop Services policy, and verified that Domain Admins were allowed.

I also confirmed the workstation was in the correct OU and validated policy application using gpupdate /force and gpresult /scope computer /r.

After verifying the configuration, I retried the connection using manually entered credentials instead of the cached credentials in Remote Desktop Connection Manager. The connection succeeded, which indicated the issue was caused by stale cached credentials.

After resolving it, I documented the troubleshooting process as a ticket with screenshots and command output so the fix could be reproduced later.
